# Product Specification

## Цель

Создать полностью клиентское SPA для глубокого изучения Kotlin. Приложение должно учить через короткое объяснение, разбор, активное восстановление, исправление ошибок, перенос навыка и интервальное повторение, а не через пассивное чтение и тесты с вариантами ответов.

## Целевая аудитория

- Новички, которые хотят освоить Kotlin последовательно и без установки окружения.
- Разработчики с опытом в других языках, которым нужен быстрый переход к идиоматике Kotlin.
- Пользователи, которым важно заниматься офлайн и сохранять прогресс без регистрации.

## Не цели продукта

- Не полноценный Kotlin compiler или online IDE.
- Не платформа с backend, аккаунтами, лидербордами и облачной синхронизацией.
- Не AI-тьютор и не генератор произвольных задач.
- Не справочник, где тема считается завершенной после прочтения.

## Основные сценарии

1. Пользователь открывает приложение и видит карту курса, текущую тему, повторения и прогресс.
2. Пользователь проходит тему через учебный цикл: объяснение, пример, упражнения разного типа, перенос, повторение.
3. Пользователь получает детерминированную обратную связь по ответу с объяснением правила проверки.
4. Пользователь возвращается позже и видит отложенные повторения.
5. Пользователь экспортирует прогресс в файл и импортирует его на другом устройстве.
6. Пользователь продолжает работу офлайн после первого открытия приложения.

## Функциональные требования

- Локальное хранение прогресса в `localStorage` через Pinia PersistedState.
- Импорт и экспорт прогресса в versioned JSON.
- Детерминированная проверка ответов без выполнения произвольного Kotlin.
- Явное отображение типа проверки для каждого задания.
- Карта курса с зависимостями между темами.
- Поиск по локальному контенту после добавления контентной базы.
- Ленивое подключение крупных разделов и редактора кода.
- Доступный интерфейс: клавиатура, фокус, ARIA через Reka UI primitives.

## Нефункциональные требования

- TypeScript strict-first: без неописанного `any`.
- Чистые функции для проверки ответов и расчета обучения.
- Предсказуемая работа без сети для core-функций.
- Малый начальный bundle; тяжелые модули подключаются лениво.
- Данные контента и состояния имеют `schemaVersion`.
- Все изменения поведения покрываются доступными проверками.

## Ограничения по Kotlin

Приложение не выполняет произвольный Kotlin-код. Проверки строятся на:

- точном сравнении;
- нормализации пробелов и форматирования;
- токенизации;
- структурных правилах;
- обязательных и запрещенных конструкциях;
- списках известных допустимых решений;
- проверке заранее заданных результатов в контролируемой области ответа.

Текст интерфейса должен отличать "проверку структуры ответа" от "запуска кода".

## Критерии качества обучения

Тема не считается освоенной, пока пользователь не:

- несколько раз ответил без подсказок;
- решил задания разных типов;
- исправил ошибочный код;
- перенес знание в новый контекст;
- успешно прошел отложенное повторение.

## Совместимость с текущим проектом

Текущий репозиторий уже содержит Vite, Vue 3, TypeScript, Tailwind CSS, Pinia PersistedState, Reka UI и Lucide Icons. Поэтому ближайшие шаги должны развивать существующий стек, а не заменять его. Роутер, PWA, поиск, тесты и контентная система добавляются отдельными этапами.
